PROJECT OVERVIEW
================================================================================

• Name: fmt/ - File Converter Web Application
• Purpose: A lightweight, privacy-focused file conversion tool that runs locally on user's machine
• Type: Full-stack web application (Frontend + Backend)
• Key Selling Point: Files are processed locally and deleted immediately after conversion (privacy-first approach)
• Supports: Video, Audio, Image, and Document (PDF) conversions


TECH STACK
================================================================================

BACKEND
-------
• Language: Python 3.10+
• Framework: Flask (lightweight, micro web framework)
• Server: Flask development server (can be deployed with Gunicorn)
• Deployment: Procfile for Heroku/cloud deployment

FRONTEND
--------
• HTML5: Semantic markup with responsive design
• CSS3: Custom grid-based styling with dark mode support
• JavaScript (Vanilla): No external frameworks - pure DOM manipulation
• Third-party JS Library: JSZip for multi-file zipping

DATABASE
--------
• None: Completely stateless application (files deleted after conversion)


LIBRARIES & DEPENDENCIES
================================================================================

PYTHON LIBRARIES
----------------

Core Web Framework:
• Flask - Web server and routing (main backend framework)
• flask-cors - Enables Cross-Origin Resource Sharing (frontend ↔ backend communication)

-----------------------

Browser APIs Used
-----------------
• FormData API - Uploading files with metadata
• Fetch API - Making HTTP POST requests
• Blob API - Handling binary file data
• File API - Accessing user-selected files
• localStorage - Saving user preferences (dark mode toggle)
• crypto.randomUUID() - Client-side unique ID generation


BACKEND ARCHITECTURE (PYTHON CODE BREAKDOWN)
================================================================================

1. CONFIGURATION & SETUP
------------------------

BASE_DIR: Absolute path to project directory
UPLOAD_FOLDER: Temporary storage for files during conversion
Format Groups: VIDEO_EXTS, AUDIO_EXTS, IMAGE_EXTS (sets of supported formats)
ALLOWED_CONVERSIONS: Dictionary mapping input format → allowed output formats
PIL_FORMAT_MAP: Pillow library's format name mapping


2. CORE FUNCTIONS
-----------------

get_ffmpeg()
• Purpose: Locate FFmpeg executable on system
• Logic: First checks system PATH, then falls back to imageio-ffmpeg bundle
• Error Handling: Raises clear error message if FFmpeg not found
• Returns: Path to FFmpeg executable

run_ffmpeg(args, timeout=600)
• Purpose: Execute FFmpeg commands for video/audio processing
• Parameters: Command arguments list + 10-minute timeout
• Error Handling: Captures stderr and raises RuntimeError with last 3 lines
• Returns: Subprocess result object

convert_file(input_path, output_format) - MAIN CONVERSION LOGIC
• Step 1: Validate conversion is allowed (checks ALLOWED_CONVERSIONS dict)
• Step 2: Generate unique output filename using UUID
• Step 3: Route to appropriate converter:
  
  Image-to-Image: Uses Pillow (PIL) library
  - Handles transparency conversion for JPEG
  - Converts color modes appropriately
  
  PDF-to-DOCX: Uses pdf2docx library
  
  Video-to-Audio: Uses FFmpeg (audio extraction only, no video re-encoding)
  
  Audio-to-Audio: Uses FFmpeg with specific codecs
  
  Video-to-Video: Uses FFmpeg with appropriate codec selection
  - MP4: libx264 codec (most compatible)
  - WebM: libvpx-vp9 codec
  - Other formats: Stream copy for speed

• Step 4: Sanity check - ensure output file exists and isn't empty
• Step 5: Return output file path


3. API ROUTES
-------------

GET @app.route("/")
• Serves the main index.html file
• Entry point for the application

GET @app.route("/api/formats")
• Returns JSON with supported formats for each category
• Used by frontend to populate dropdown menus
• Format: {video: {from: [...], to: [...]}, ...}

POST @app.route("/api/convert")
Receives: 
  - file: Multipart form-data file upload
  - format: Target format as string
  - quality (optional): For image quality

Process:
  1. Validate file exists
  2. Secure filename using werkzeug
  3. Generate unique temporary filepath
  4. Call convert_file() function
  5. Send converted file back to browser
  6. Auto-cleanup both input and output files (using @after_this_request decorator)

Error Handling: Returns JSON errors with appropriate HTTP status codes
  - 400: Bad request (no file, no format)
  - 504: Timeout (file too large)
  - 500: Conversion failed


4. SECURITY & PRIVACY FEATURES
------------------------------

• secure_filename() - Sanitizes uploaded filenames
• uuid.uuid4() - Prevents filename guessing/enumeration
• File cleanup after conversion - Automatic deletion (privacy-first)
• @after_this_request decorator - Ensures cleanup even on errors
• try/finally block - Guarantees input file deletion
• CORS enabled - Only allows frontend requests
• No database - No user data retention

DATA FLOW (COMPLETE REQUEST CYCLE)
================================================================================

1. User drags/drops files into browser
2. JavaScript reads File objects from input
3. Creates job entries (id, file, format, etc.)
4. Renders UI cards for each file
5. User clicks "Convert All"
6. For each job:
   - Creates FormData with file + target format
   - Sends POST to /api/convert
   - Shows progress: "Uploading..." → "Converting..." → "Done"
7. Backend receives multipart form data
8. Validates file and format
9. Calls convert_file() with appropriate processor
10. Returns converted file as blob
11. Frontend receives blob, stores in job.blob
12. Shows download button
13. Backend auto-cleans temporary files
14. User clicks download
15. Browser saves file using triggerDownload()

DEPLOYMENT
================================================================================

• Procfile: web: python app.py (for Heroku/cloud platforms)
• Environment Variable: PORT (defaults to 5000 if not set)
• Server Mode: host="0.0.0.0" - accessible from any network interface
• Threading: threaded=True - handles concurrent requests

How to Run Locally:
1. Create virtual environment: python -m venv venv
2. Activate: .\venv\Scripts\Activate.ps1 (Windows) or source venv/bin/activate (Linux/Mac)
3. Install dependencies: pip install -r requirements.txt
4. Install FFmpeg: winget install ffmpeg (Windows) or brew install ffmpeg (Mac)
5. Run app: python app.py
6. Visit: http://localhost:5000/
